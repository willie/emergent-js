## 2024-06-13 - [Sanitization Bypass and Type Error in API Routes]
**Vulnerability:** Arbitrary `.json` file creation/deletion/read when input path sanitization resolves to an empty string. The Storage API endpoint also threw unhandled TypeErrors when user input parsed from JSON body was not an expected string (e.g. object, array), resulting in a 500 error and stack trace.
**Learning:** Empty string check is required after stripping invalid characters using a regex, as a completely invalid key string (e.g., `<script>`) gets replaced into an empty string, causing the application to construct paths like `.json` at the root of `data`. Additionally, parsed JSON payloads from `req.json()` must have their types validated before invoking string methods to prevent 500 errors or unhandled exceptions.
**Prevention:** After replacing characters for a `key` or `filepath`, enforce `if (!cleanKey)` to prevent empty path vulnerabilities. Before invoking methods like `.replace()`, validate the input type e.g., `if (typeof key !== 'string')`.
